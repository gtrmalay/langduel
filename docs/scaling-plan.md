# Scaling Plan (LangDuel MVP -> Production)

## Цель
Показать понятный план масштабирования без оверинжиниринга.

## Этап 0: MVP (сейчас)
- In-memory комнаты и игроки (`duel.Manager`).
- Один процесс сервера.
- WebSocket внутри одного инстанса.
- Нет БД, нет авторизации.

## Этап 0.5: UX и базовый визуал (быстро и просто)
- Минимальная анимация боя (удар/вспышка у получателя урона).
- Таймер раунда на клиенте.
- Экран “ожидание соперника”.
- Простой экран результата дуэли.

## Этап 1: Стабильность и мониторинг
- Метрики (кол-во комнат, активных WS, средний RTT).
- Логи с уровнями.
- Простейшие тесты на `duel`.
- Ограничение размера входящих сообщений.

## Этап 2: База данных
- Ввести интерфейс `Storage`:
  - `CreateMatch`
  - `SaveRound`
  - `SaveAnswer`
- Реализация на PostgreSQL.
- Внести миграции.

## Этап 2.5: Функциональные use-case (без усложнений)
- Гость (ник) / зарегистрированный пользователь.
- Создание вызова + подключение к вызову.
- Выбор темы/языка.
- Генерация фраз по теме.
- Просмотр результата дуэли.
- Простая статистика (wins/losses).

## Опционально: ИИ-генерация фраз
- Источник фраз на основе выбранной темы/языка.
- Использовать как подключаемый модуль, чтобы можно было отключить.
- Сначала оставить ручные наборы, ИИ добавить позже.

## Этап 3: Горизонтальное масштабирование
Проблема: WebSocket состояние живет в памяти одного процесса.

Решение:
- Вынести состояния комнат в Redis.
- Использовать Redis Pub/Sub или NATS для рассылки событий.
- Каждый инстанс сервера читает события своей комнаты.

## Этап 4: Балансировка
- Load balancer (nginx/traefik).
- Sticky sessions для WS (или внешний state + pub/sub).
- Health-check эндпоинт.

## Этап 5: Безопасность
- Ограничить `CheckOrigin`.
- Rate limit на join/answer.
- Авторизация пользователя.
- Валидация входящих данных.

## Этап 6: Производительность
- Бенчмарки `duel.Manager`.
- Оптимизация сериализации событий.
- Ограничение на размер комнат и время жизни.

## Этап 7: Гейм-мета (если останется время)
- Скины и косметика.
- Внутриигровая валюта за матчи.
- Достижения (ачивки).
- Рейтинг (ELO или простая таблица).

## Небольшой чек-лист перед продом
- Метрики и алерты.
- Backups БД.
- Логи и трассировки.
- Документация API.
